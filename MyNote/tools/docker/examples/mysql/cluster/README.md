两种集群方式：

- Replication：速度快，弱一致性，价值低，日志，新闻，帖子
- PXC：速度慢，强一致性，高价值，订单，账户，财务

| Replication                                                  | PXC                                                          |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| 数据同步是单向的，master负责写，然后异步复制给slave；如果slave写入数据，不会复制给master。 | 数据同步时双向的，任何一个mysql节点写入数据，都会同步到集群中其它的节点。 |
| 异步复制，从和主无法保证数据的一致性                         | 同步复制，事务在所有集群节点要么同时提交，要么同时不提交     |
